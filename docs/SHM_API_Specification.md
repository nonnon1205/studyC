# 共有メモリ (SHM) API 仕様書

## 1. 概要
プロセス間で共有メモリを用いた高速なデータ共有を行うためのAPIインターフェース仕様を定義します。
本APIは、システム内での状態コード (`status_code`) およびメッセージ文字列 (`message`) の共有と排他制御を隠蔽し、安全なアクセスを提供します。

本プロジェクトでは学習および移植性の観点から、**System V 共有メモリ (`shmget`/`shmat`)** と **POSIX 共有メモリ (`shm_open`/`mmap`)** の2種類のバックエンド実装を提供しています。APIのインターフェース (`shm_api.h`) は完全に統一されており、利用側のコードを変更することなく、ビルド時（`Makefile`の変数）の設定で実装を透過的に切り替えることが可能です。

## 2. アーキテクチャと重要な制約 (Contract)

### 2.1. 固定長バッファの前提
共有メモリ上で扱えるメッセージ文字列の最大長は、コンパイル時に固定長（例: 256バイト）として `SharedData` 構造体内に定義されています。
動的なサイズチェックによるオーバーヘッドを避けるため、本APIは以下の契約に基づいて動作します。

- **読み出し側 (利用側) の責任:**
  - `shm_api_read` を呼び出す際、引数 `out_msg` には**必ず共有バッファの最大長以上のサイズを持つメモリ領域を確保して渡す**責任があります。
  - 小さすぎるバッファを渡した場合、メモリ破壊（Write Out-of-Bounds）が発生する危険性があります。

- **API側の責任と保護範囲:**
  - API内部では呼び出し元から渡されたバッファサイズの検証は行いません。
  - ただし、共有メモリ上のデータが破損しておりNULL終端 (`\0`) が失われている最悪のケースを想定し、APIは「共有バッファの最大長」以上の読み取りは行いません（Read Out-of-Boundsの防止）。

## 3. APIリファレンス

### 3.1. 初期化と接続
```c
ShmHandle shm_api_init(void);
```
- **機能:** 共有メモリセグメントを作成、または既存のセグメントに接続し、操作用の不透明ハンドルを返します。
- **戻り値:** 成功時はハンドル(`ShmHandle`)、失敗時は `NULL`。
- **補足:** プロセスの中で最初に呼び出したものが「作成者」として認識され、メモリサイズの確保および共有Mutexの初期化を自動的に行います。

### 3.2. データ書き込み
```c
bool shm_api_write(ShmHandle handle, int status, const char* msg);
```
- **機能:** 共有メモリに状態コードとメッセージを排他制御付きで書き込みます。
- **引数:**
  - `handle`: `shm_api_init` で取得したハンドル
  - `status`: 任意のステータスコード
  - `msg`: 書き込むメッセージ文字列。**`NULL` を渡すことは許可されません**（未定義動作防止のため `assert` で弾かれます）。
- **戻り値:** 成功時は `true`、失敗時は `false`。
- **補足:** `msg` は共有メモリの最大サイズに安全に切り詰められて書き込まれます。

### 3.3. データ読み出し
```c
bool shm_api_read(ShmHandle handle, int* out_status, char* out_msg);
```
- **機能:** 共有メモリから現在の状態コードとメッセージを排他制御付きで読み出します。
- **引数:**
  - `handle`: APIハンドル
  - `out_status`: ステータスコードを格納するポインタ（不要な場合は `NULL` 可）
  - `out_msg`: メッセージ文字列を格納するバッファ（不要な場合は `NULL` 可）
- **重大な注意:** `out_msg` には、必ず共有バッファの最大長（256バイト）以上のメモリを渡してください。

### 3.4. 終了処理
```c
void shm_api_close(ShmHandle handle);
void shm_api_destroy(ShmHandle handle);
```
- **shm_api_close:** プロセス単位で共有メモリから切断し、メモリマッピングを解除します。プロセス終了時に呼び出します。
- **shm_api_destroy:** 共有メモリから切断し、自身が「作成者」であった場合は**システム全体から共有メモリ領域を完全に破棄**します。システム全体の終了・クリーンアップ時に呼び出します。
