# API仕様書

## 1. 概要
本ドキュメントは、システムを構成する各プロセス間で利用される通信インターフェース、および外部管理クライアント向けに提供される管理APIの仕様を定義します。

## 2. プロセス間通信 (IPC) 仕様

### 2.1. データストリーム通信 (共有メモリ + メッセージキュー)
高スループットなデータの受け渡しを行うためのインターフェースです。

- **送信元 (Producer):** PollIO などのデータ生産者
- **受信元 (Consumer):** TestMsgRcv
- **方式:** System V 共有メモリ (SHM) + メッセージキュー (MQ)
- **通知用キー:** `SYSTEM_IPC_KEY`
- **処理フロー:**
  1. Producerが共有メモリの所定の領域に実データを書き込む。
  2. ProducerがMQ (`SYSTEM_IPC_KEY`) に更新通知メッセージを送信する。
  3. ConsumerがMQで通知をブロック待機し、受信後に共有メモリから実データを読み出す。

### 2.2. 整形データ転送 (TCPソケット)
TestMsgRcvで処理・集約されたデータを最終的なビューアに送信するためのインターフェースです。

- **送信元:** TestMsgRcv
- **受信元:** TestUDPKill
- **方式:** TCPストリーム通信
- **処理フロー:**
  1. TestUDPKillが内部の `tcp_worker` にてTCP接続を待ち受ける (LISTEN)。
  2. TestMsgRcvがTestUDPKillに対してTCP接続を確立する。
  3. TestMsgRcvは共有メモリ等から取得したデータをTCP経由でTestUDPKillへ継続的に送信(`send`)する。

## 3. 管理機能API (UNIXドメインソケット)

### 3.1. 概要
稼働中の各プロセス（TestMsgRcv, TestUDPKill）に対して、外部から動的に設定変更やステータス取得を行うための管理プレーンインターフェースです。

### 3.2. エンドポイント仕様
- **通信方式:** UNIXドメインソケット (データグラム通信: `SOCK_DGRAM`)
- **ソケットパス:** `MGMT_SOCKET_PATH` (対象プロセスごとに定義されるパス)

### 3.3. データ構造
#### リクエスト: MgmtCommandRequest
クライアントから対象プロセスへ送信されるコマンド要求構造体。
- `target_module`: 操作対象の内部モジュール名
- `cmd_type`: 実行するコマンドの種別 (`MGMT_CMD_*`)
- `payload`: コマンド固有のパラメータ情報

#### レスポンス: MgmtCommandResponse
対象プロセスからクライアントへ返却されるコマンド実行結果構造体。
- `result_code`: 実行結果ステータス (`MGMT_RESULT_*`)
- `payload`: 実行結果の詳細データ（メトリクス取得時はJSON形式のスナップショットなど）

### 3.4. サポートする主なコマンド (`MGMT_CMD_*`)
現状の共通管理ライブラリで提供される代表的なコマンドです。

1. **ログレベル変更:** 稼働中にシステムのログ出力レベルを動的に変更する。
2. **メトリクス取得:** 各プロセスで収集されたパフォーマンスメトリクス（レイテンシ、処理数、エラー数など）を取得する。
3. **設定変更:** プロセス再起動なしに、対象モジュール内部の動作パラメータを更新する。
